# 信息收集清单

这些清单只用于确实需要个性化判断的场景。
不要把它们强行套到简单事实查询或纯知识解释里。

## 个性化投资建议的最小画像

在给出方向性建议前，至少收集这些信息：

- 投资目标：增值、现金流、保值、对冲，或其他
- 持有周期：几天、几个月、几年
- 风险承受能力：低、中、高，或用户自己的描述
- 当前暴露：已有仓位或计划仓位占比
- 约束条件：流动性需求、杠杆限制、集中度限制、税务或账户限制

如果缺了其中一项，就明确指出，并选择“补一个简短问题”或“先给条件式回答”。

## 单一标的的买卖判断

最少需要的信息：

- 标的名称或代码
- 预期持有周期
- 这个标的在组合里的角色
- 当前仓位或计划仓位
- 风险承受能力

有帮助但不是必须的信息：

- 成本价
- 可接受的最大回撤
- 正在比较的替代选项

如果用户不愿意补充背景：

- 不要给直接的个性化推荐。
- 改为提供情景化判断框架。

## 投资组合诊断

最少需要的信息：

- 持仓列表
- 大致权重
- 现金需求或短期负债
- 投资目标
- 风险承受能力
- 调仓频率或调仓灵活度

诊断时优先看：

- 集中度风险
- 隐含风格押注
- 流动性与现金需求错配
- 分散化质量
- 与投资目标是否一致

## 基金筛选

最少需要的信息：

- 想要的暴露方向：宽基、行业、因子、债券久期、红利等
- 持有周期
- 对费率是否敏感
- 是否在意税务差异
- 风险承受能力

比较时至少覆盖：

- 暴露是否匹配需求
- 费率拖累
- 流动性或跟踪质量
- 集中度或编制方法差异

## 信息不完整时的处理原则

当信息不完整时：

1. 先回答可以安全回答的部分。
2. 点出最影响结论的 1 到 3 项缺失信息。
3. 说明这些信息会怎样改变判断。
4. 不要把一般性回答伪装成量身定制的建议。
